Guidelines


Submission Guidelines



For you to be considered a valid submission we require that you be apart of a project.



Your project will need the following inside the Project View:


  • A description of the project
  • A public git repository where we can see your codes and commits (Github, Gitlab, Anything that is publicly viewable).
  • A shared url for yours slides which we can view (Google Slides / PowerPoint).

The description of the project should answer these questions:


  • What was the goal of the project?
  • What does the project do?
  • How and what was it build with?
  • What were the technical difficulties that you encountered?
  • What improvements could you make in the future (if you think youâ€™d work on this more)

The public git repository should have these:


  • Photo or Video in ReadME to see App
  • All the code you wrote in the project
  • A url to the demo website if deployed (Optional)
  • A url to the demo website if deployed (Optional)Good documentation KEKW (Optional)

The presentation slides isn't strict, please emphasize on the demo of your project, we want to see the thing! That side, here are some suggestions of other things you can include:


  • Title
  • Introduction Slide (Where you introduce yourselves)
  • Goal of the project slide (Why did you make it)
  • Stack / Frameworks / Technology used stack slide
  • Demo (just a way for you to segue to your demo)


Judging Format



The presentation slides isn't strict, please emphasize on the demo of your project, we want to see the thing! That side, here are some suggestions of other things you can include:



(1) Offline Judging based on these criteria, we'll pick the top 25 after going through all your submitted Github repo links (README, code, etc.) and slides.


  • Meme: How much of a fun factor does your Project have (8)
  • Originality: How original/impressive is your project?(10)
  • Technical Difficulty: How difficult was your project (8)
  • Utility: How useful is it? (8)

(2) On stream Judging - Top 25


  • Each participant will have 4 minutes to present, and the judges will discuss/ask questions for 1 minute.
  • Each judge will say their personal scores of what they thought of their project out of 5 with chat scoring with a poll between 1-5 too where we average out the scores.
  • Best Meme and Social hacks will be chosen at the end by the hosts!
